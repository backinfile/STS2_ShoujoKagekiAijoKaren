# 卡牌导出数据格式

本文档描述 `karen exportcards` 命令生成的输出结构，用于基于网页的卡牌图鉴展示。

## 文件目录结构

导出命令会在指定目录（默认 `user://KarenCardExports/`）生成以下文件：

```
KarenCardExports/
├── cards.json                          # 所有卡牌的元数据索引
├── KAREN_STRIKE_base.png               # 卡牌本体
├── KAREN_STRIKE_upgraded.png           # 卡牌升级版（仅可升级卡牌有）
├── KAREN_DEFEND_base.png               # 另一张卡牌本体
├── KAREN_DEFEND_upgraded.png
├── KAREN_AQUARIUM_CARD_base.png
├── KAREN_AQUARIUM_CARD_upgraded.png
└── ...                                 # CardHoverTip 引用的其他卡牌 base 图
```

### 图片命名规则

| 图片类型 | 命名格式 | 示例 |
|---------|---------|------|
| 卡牌本体 | `{卡牌ID}_base.png` | `KAREN_STRIKE_base.png` |
| 卡牌升级版 | `{卡牌ID}_upgraded.png` | `KAREN_STRIKE_upgraded.png` |
| CardHoverTip 引用卡 | `{引用卡牌ID}_base.png` | `KAREN_DEFEND_base.png` |

卡牌 ID 中的非法文件名字符会被替换为下划线 `_`。

---

## JSON 格式（cards.json）

```json
{
  "cards": [
    {
      "id": "KAREN_AQUARIUM_CARD",
      "name": "水族馆",
      "cost": "1",
      "rarity": "罕见",
      "type": "能力",
      "tips": [
        {
          "type": "text",
          "title": "约定牌堆",
          "description": "爱城华恋特有的牌堆。"
        },
        {
          "type": "card",
          "cardId": "KAREN_DEFEND",
          "image": "KAREN_DEFEND_base.png"
        }
      ],
      "images": {
        "base": "KAREN_AQUARIUM_CARD_base.png",
        "upgraded": "KAREN_AQUARIUM_CARD_upgraded.png"
      }
    }
  ]
}
```

## 字段说明

### CardExportInfo（单张卡牌）

| 字段 | 类型 | 说明 |
|-----|------|------|
| `id` | `string` | 卡牌唯一标识，如 `KAREN_STRIKE` |
| `name` | `string` | 卡牌中文名称（来自本地化 Title） |
| `cost` | `string` | 费用。可能值为 `"0"`~`"3"`、`"X"`、 `""`（不消耗） |
| `rarity` | `string` | 稀有度中文。见下表 |
| `type` | `string` | 类型中文。见下表 |
| `tips` | `TipInfo[]` | 悬浮提示列表，顺序与原卡牌一致 |
| `images` | `ImageInfo` | 卡图文件名索引 |

### 稀有度取值

| 值 | 对应 `CardRarity` |
|---|------------------|
| 基础 | `Basic` |
| 普通 | `Common` |
| 罕见 | `Uncommon` |
| 稀有 | `Rare` |
| 远古 | `Ancient` |
| 事件 | `Event` |
| 衍生 | `Token` |
| 状态 | `Status` |
| 诅咒 | `Curse` |
| 任务 | `Quest` |

### 类型取值

| 值 | 对应 `CardType` |
|---|----------------|
| 攻击 | `Attack` |
| 技能 | `Skill` |
| 能力 | `Power` |
| 状态 | `Status` |
| 诅咒 | `Curse` |

### TipInfo（悬浮提示）

`type` 字段决定提示类型：

**`type: "text"`** — 文本提示

| 字段 | 类型 | 说明 |
|-----|------|------|
| `type` | `string` | 固定为 `"text"` |
| `title` | `string?` | 提示标题，可能为 `null` |
| `description` | `string?` | 提示描述内容，可能为 `null` |

**`type: "card"`** — 卡牌引用提示

| 字段 | 类型 | 说明 |
|-----|------|------|
| `type` | `string` | 固定为 `"card"` |
| `cardId` | `string?` | 被引用卡牌的 ID |
| `image` | `string?` | 被引用卡牌的 `base` 图片文件名 |

### ImageInfo（卡图索引）

| 字段 | 类型 | 说明 |
|-----|------|------|
| `base` | `string` | 卡牌本体图片文件名 |
| `upgraded` | `string?` | 升级版图片文件名。不可升级的卡牌此字段为 `null` |

## 网页开发建议

### 文件加载

将 `cards.json` 和所有 `.png` 放在同一目录下（或子目录中保持相对路径），通过 `fetch("cards.json")` 读取索引，再按需加载图片。

### 卡牌列表页

遍历 `cards` 数组，每张卡牌展示：
- 左侧/上方：`images.base` 对应的卡图
- 卡牌名称 + 费用 + 稀有度 + 类型标签
- 点击后跳转到详情页

### 卡牌详情页

根据 `id` 查找对应卡牌数据：
- 并排展示 `images.base` 和 `images.upgraded`（如果存在）
- 下方渲染 `tips` 列表：
  - `text` 类型：渲染为带标题的文本框
  - `card` 类型：渲染为缩略图卡片，图片路径为 `tip.image`

### 去重说明

CardHoverTip 引用的卡牌图片会在所有 Karen 卡牌间去重。例如 3 张不同的 Karen 卡牌都引用了 `KAREN_DEFEND`，输出目录中只会生成一张 `KAREN_DEFEND_base.png`。JSON 中各 `card` 类型的 `image` 字段都指向同一文件。
