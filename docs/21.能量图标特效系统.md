# 能量图标特效系统（Energy Counter / VFX）

> 基于游戏本体反编译代码（`spire-codex/extraction/raw/`）整理，供 Mod 项目参考。

---

## 1. 整体架构

每个可玩角色拥有独立的**能量计数器**（Energy Counter），由以下组件构成：

| 组件 | 类型 | 作用 |
|------|------|------|
| `NEnergyCounter` | `Control`（C# 脚本） | 核心逻辑类，处理能量变化、悬浮提示、动画 |
| `{角色}_energy_counter.tscn` | PackedScene | 场景文件，定义贴图层、粒子、Label 等节点结构 |
| `{角色}_energy_vfx_back.tscn` | PackedScene | 能量球**背后**的常驻粒子特效 |
| `{角色}_energy_vfx_front.tscn` | PackedScene | 能量球**前方**的常驻粒子特效 |
| `NParticlesContainer` | `Node2D`（C# 脚本） | 粒子容器，批量控制 `GPUParticles2D` 的 `Emitting` 和 `Restart` |

## 2. 角色配置

### EnergyCounterPath 自动计算

在 `CharacterModel.cs` 中，`EnergyCounterPath` 由角色 ID 自动推导：

```csharp
public string EnergyCounterPath =>
    SceneHelper.GetScenePath("combat/energy_counters/" + base.Id.Entry.ToLowerInvariant() + "_energy_counter");
```

**示例**：
- Ironclad → `res://scenes/combat/energy_counters/ironclad_energy_counter.tscn`
- Silent → `res://scenes/combat/energy_counters/silent_energy_counter.tscn`
- Karen（Mod）→ `res://scenes/combat/energy_counters/karen_energy_counter.tscn`

### 创建方式

```csharp
public static NEnergyCounter? Create(Player player)
{
    NEnergyCounter counter = PreloadManager.Cache
        .GetScene(player.Character.EnergyCounterPath)
        .Instantiate<NEnergyCounter>(PackedScene.GenEditState.Disabled);
    counter._player = player;
    return counter;
}
```

## 3. energy_counter 场景结构（所有角色统一）

所有角色的能量计数器场景采用**完全一致的节点层级**，仅贴图（`Texture2D`）和 vfx 场景路径不同。

### 节点层级

```
{角色}EnergyCounter (Control)  ← script = NEnergyCounter.cs
├── BurstBack (CPUParticles2D)      ← 能量增加时的背面爆发粒子
├── EnergyVfxBack (Node2D)          ← instance of {角色}_energy_vfx_back.tscn
│                                    ← unique_name_in_owner = true
├── Layers (Control)
│   ├── Layer1 (TextureRect)         ← 基础能量球贴图（darkened material）
│   ├── RotationLayers (Control)     ← 旋转层容器
│   │   ├── Layer2 (TextureRect)     ← 逆时针慢速旋转
│   │   └── Layer3 (TextureRect)     ← 顺时针快速旋转
│   ├── Layer4 (TextureRect)         ← 可见性由能量状态控制
│   └── Layer5 (TextureRect)         ← 同上
├── BurstFront (CPUParticles2D)     ← 能量增加时的正面爆发粒子
├── EnergyVfxFront (Node2D)         ← instance of {角色}_energy_vfx_front.tscn
│                                    ← unique_name_in_owner = true
└── Label (Label)                    ← 能量数值文本 "3/3"
    └── script = MegaLabel.cs (addons/mega_text)
```

### 关键节点说明

| 节点名 | `unique_name_in_owner` | NEnergyCounter 中的获取方式 |
|--------|----------------------|---------------------------|
| `Label` | 否 | `GetNode<MegaLabel>("Label")` |
| `Layers` | **是** | `GetNode<Control>("%Layers")` |
| `RotationLayers` | **是** | `GetNode<Control>("%RotationLayers")` |
| `EnergyVfxBack` | **是** | `GetNode<NParticlesContainer>("%EnergyVfxBack")` |
| `EnergyVfxFront` | **是** | `GetNode<NParticlesContainer>("%EnergyVfxFront")` |

### 贴图资源命名规范

```
images/ui/combat/energy_counters/{角色id小写}/{角色id小写}_orb_layer_{1-5}.png
```

**示例**：
- `images/ui/combat/energy_counters/ironclad/ironclad_orb_layer_1.png`
- `images/ui/combat/energy_counters/silent/silent_orb_layer_3.png`
- `images/ui/combat/energy_counters/karen/karen_orb_layer_2.png`

### 通用爆发贴图

```
images/ui/combat/energy_counters/energy_burst/energy_orb_shine.png   ← BurstBack
images/ui/combat/energy_counters/energy_burst/energy_orb_burst.png   ← BurstFront
```

## 4. energy_vfx 场景结构（每个角色独立）

每个角色的能量特效分为 **back**（底层光晕）和 **front**（表层粒子）。

### 通用结构

```
{角色}_energy_vfx_back.tscn / {角色}_energy_vfx_front.tscn
└── root (Node2D)  ← script = NParticlesContainer.cs
                     node_paths=PackedStringArray("_particles")
                     _particles = [NodePath("粒子节点名")]
    └── {粒子节点} (GPUParticles2D)  ← 可以多个
```

### NParticlesContainer 的作用

```csharp
public partial class NParticlesContainer : Node2D
{
    [Export]
    private Array<GpuParticles2D>? _particles;

    public void SetEmitting(bool emitting)  // 批量开关
    public void Restart()                   // 批量重启（触发 OnEnergyChanged）
}
```

### 各角色差异对比

| 角色 | vfx_back | vfx_front | 特效风格 |
|------|----------|-----------|----------|
| **Ironclad** | 1x `vfx_common_glow`（橙色光晕） | 5x 粒子（火焰翻书×3 + 余烬 + 极光环） | 火焰/燃烧 |
| **Silent** | 1x `vfx_common_glow`（绿色光晕） | — | 毒/绿色 |
| **Defect** | 1x `vfx_common_glow`（蓝色光晕） | — | 冰冷/科技 |
| **Regent** | — | — | — |
| **Necrobinder** | — | — | — |

> **注**：Silent/Defect 的 vfx_back 结构与 Ironclad 几乎一致，仅 `self_modulate` 颜色不同。

### vfx_back 通用模板

```tscn
[ext_resource type="Script" path="res://src/Core/Nodes/Vfx/Utilities/NParticlesContainer.cs" id="1_xxx"]
[ext_resource type="PackedScene" path="res://scenes/vfx/common/vfx_common_glow.tscn" id="2_xxx"]

# SubResource: Curve + CurveTexture + ParticleProcessMaterial

[node name="{角色}_energy_vfx_back" type="Node2D" node_paths=PackedStringArray("_particles")]
script = ExtResource("1_xxx")
_particles = [NodePath("vfx_common_glow")]

[node name="vfx_common_glow" parent="." instance=ExtResource("2_xxx")]
self_modulate = Color(角色主题色)    ← 仅此处不同
process_material = SubResource("ParticleProcessMaterial_xxx")
```

## 5. NEnergyCounter 核心逻辑

### 能量变化响应

```csharp
private void OnEnergyChanged(int oldEnergy, int newEnergy)
{
    if (oldEnergy < newEnergy)
    {
        _backVfx?.Restart();   // 能量增加时重启背后粒子
        _frontVfx?.Restart();  // 能量增加时重启前方粒子
    }
}
```

> **注意**：只有**能量增加**时触发粒子重启，减少时不触发。

### 能量耗尽暗化

```csharp
private void OnCombatStateChanged(CombatState combatState)
{
    // 能量为 0 时：
    // 1. 所有 RotationLayers 子节点材质替换为 energy_orb_dark.tres
    // 2. Layers.Modulate = DarkGray
    // 3. 旋转速度降为 5°/s（正常 30°/s）
}
```

### _Process 旋转动画

```csharp
public override void _Process(double delta)
{
    float speed = (_player.PlayerCombatState.Energy == 0) ? 5f : 30f;
    for (int i = 0; i < _rotationLayers.GetChildCount(); i++)
    {
        _rotationLayers.GetChild<Control>(i).RotationDegrees += delta * speed * (i + 1);
    }
}
```

> Layer2 和 Layer3 以不同速度反向旋转，形成能量球的动态效果。

## 6. NParticlesContainer 的已知问题

### 问题：`Failed to get property '_particles'`

Godot 加载 `.tscn` 时，序列化系统无法正确访问跨程序集的**私有**导出字段：

```csharp
// 游戏本体代码（在 sts2.dll 中）
[Export]
private Array<GpuParticles2D>? _particles;   // 私有字段，Mod 无法序列化
```

### 解决方案（Mod 项目）

**方案 A：移除 `node_paths`（推荐，最轻量）**

```tscn
# 删除 node_paths 和 _particles 属性赋值
[node name="karen_energy_vfx_back" type="Node2D"]
script = ExtResource("1_karen")
# 不再声明 _particles
```

然后在 C# 脚本中 `_Ready()` 时手动查找子节点：

```csharp
public override void _Ready()
{
    _particles = new Array<GpuParticles2D>();
    foreach (var child in GetChildren())
    {
        if (child is GpuParticles2D p)
            _particles.Add(p);
    }
}
```

**方案 B：直接继承 Node2D（放弃 NParticlesContainer）**

将 `KarenParticlesContainer` 改为直接继承 `Node2D`，在类内自行声明 `_particles` 和操作方法。这样 Godot 不会尝试序列化基类的私有字段。

**方案 C：删除 vfx 场景（极简方案）**

如果不需要粒子特效，可直接从 `karen_energy_counter.tscn` 中删除 `EnergyVfxBack` 和 `EnergyVfxFront` 节点，并删除对应的 vfx 场景文件。这样能量增加时不会有粒子特效，但功能完全正常。

## 8. Mod 项目注意事项

1. **不需要创建 `SNEnergyCounter` 或 `SMegaLabel` 包装类**：游戏本体中不存在这些类，它们是 Mod 项目为了绕过 `[GlobalClass]` 注册冲突而创建的。正确做法是直接使用原版类，或在子类中去掉 `[GlobalClass]`。

2. **不需要复制 `MegaLabel.cs`**：`MegaLabel` 已在游戏本体 `sts2.dll` 中，复制到 `addons/mega_text/` 会导致 CS0436 类型冲突。

3. **能量计数器场景必须与角色 ID 匹配**：`karen_energy_counter.tscn` 中的根节点名应为 `KarenEnergyCounter`，但这不影响运行时逻辑（NEnergyCounter 通过 `GetNode` 按路径查找子节点）。

4. **最小可行方案**：如果不需要粒子特效，可以：
   - 保留 `karen_energy_counter.tscn`（删除 EnergyVfxBack/Front 引用）
   - 删除 `scenes/vfx/energy/karen/` 目录
   - 删除 `KarenParticlesContainer.cs`
   - 能量计数器仍能正常显示数值、贴图和旋转动画
