# Karen 能量计数器（Energy Counter）实现文档

## 概述

为 Karen 角色定制的能量计数器，继承自 `NEnergyCounter`，使用珊瑚红（`#FB5458`）主题特效。核心方案是通过 Harmony Patch 桥接父类的能量变化事件到子类自定义的粒子特效节点。

## 核心逻辑

### 1. SKEnergyCounter

**路径**：`src/Core/Node/SKEnergyCounter.cs`

继承 `NEnergyCounter`，覆盖 `_Ready()`：

1. 调用 `base._Ready()` 完成父类初始化（Label、Layers、HoverTip 等）
2. 将父类的 `_backVfx` / `_frontVfx` 字段置 `null`，防止后续 `?.Restart()` 时引用不存在的节点
3. 通过 `%MyEnergyVfxBack` / `%MyEnergyVfxFront` 获取自定义粒子容器

```csharp
public partial class SKEnergyCounter : NEnergyCounter
{
    public SNParticlesContainer _myBackVfx;
    public SNParticlesContainer _myFrontVfx;

    public override void _Ready()
    {
        base._Ready();
        AccessTools.Field(typeof(NEnergyCounter), "_backVfx")?.SetValue(this, null);
        AccessTools.Field(typeof(NEnergyCounter), "_frontVfx")?.SetValue(this, null);

        _myBackVfx = GetNode<SNParticlesContainer>("%MyEnergyVfxBack");
        _myFrontVfx = GetNode<SNParticlesContainer>("%MyEnergyVfxFront");
    }
}
```

**注意**：`_myBackVfx` / `_myFrontVfx` 必须是 `public`，因为 Patch 在另一个程序集中访问。

### 2. SNParticlesContainer

**路径**：`src/Core/Node/SNParticlesContainer.cs`

不继承 `NParticlesContainer`，直接继承 `Node2D`。运行时遍历子节点收集 `GpuParticles2D`，提供 `SetEmitting()` 和 `Restart()` 接口。

```csharp
public partial class SNParticlesContainer : Node2D
{
    private Array<GpuParticles2D> _particles = new();

    public override void _Ready()
    {
        foreach (var child in GetChildren())
        {
            if (child is GpuParticles2D p)
                _particles.Add(p);
        }
    }

    public void SetEmitting(bool emitting) { ... }
    public void Restart() { ... }
}
```

### 3. NEnergyCounterReadyPatch（IL Transpiler）

**路径**：`src/Core/Patches/NEnergyCounterReadyPatch.cs`

**问题**：`NEnergyCounter._Ready()` 中会执行：
```csharp
_backVfx = GetNode<NParticlesContainer>("%EnergyVfxBack");
_frontVfx = GetNode<NParticlesContainer>("%EnergyVfxFront");
```

Karen 场景中没有这两个节点，且 `NParticlesContainer` 类型也不兼容 `SNParticlesContainer`，会导致运行时抛异常。

**方案**：Transpiler 将 `GetNode<NParticlesContainer>()` 替换为 `GetNodeOrNull<NParticlesContainer>()`，找不到节点时返回 `null` 而不是抛异常。

```csharp
[HarmonyPatch(typeof(NEnergyCounter), "_Ready")]
public class NEnergyCounterReadyPatch
{
    static IEnumerable<CodeInstruction> Transpiler(IEnumerable<CodeInstruction> instructions)
    {
        foreach (var code in instructions)
        {
            if (code.opcode == OpCodes.Call && code.operand is MethodInfo mi && mi == _getNodeMethod)
            {
                code.operand = _getNodeOrNullMethod;
            }
            yield return code;
        }
    }
}
```

### 4. NEnergyCounterOnEnergyChangedPatch（Postfix）

**路径**：`src/Core/Patches/NEnergyCounterOnEnergyChangedPatch.cs`

**问题**：`NEnergyCounter.OnEnergyChanged` 中：
```csharp
private void OnEnergyChanged(int oldEnergy, int newEnergy)
{
    if (oldEnergy < newEnergy)
    {
        _backVfx?.Restart();
        _frontVfx?.Restart();
    }
}
```

父类字段已被置 `null`，不会触发 Karen 的自定义特效。

**方案**：Postfix Patch 在父类方法执行后，检测实例是否为 `SKEnergyCounter`，若是则调用其 `_myBackVfx?.Restart()` / `_myFrontVfx?.Restart()`。

```csharp
[HarmonyPatch(typeof(NEnergyCounter), "OnEnergyChanged")]
public class NEnergyCounterOnEnergyChangedPatch
{
    private static void Postfix(NEnergyCounter __instance, int oldEnergy, int newEnergy)
    {
        if (oldEnergy >= newEnergy) return;
        if (__instance is SKEnergyCounter counter)
        {
            counter._myBackVfx?.Restart();
            counter._myFrontVfx?.Restart();
        }
    }
}
```

## 场景结构

### karen_energy_counter.tscn

```
KarenEnergyCounter (Control, SKEnergyCounter)
├── BurstBack (CPUParticles2D)           ← 旧 burst 特效，保留
├── MyEnergyVfxBack (SNParticlesContainer) ← 能量增加时背景 glow
│   └── Glow (GPUParticles2D)
├── Layers (Control)
│   ├── Layer1~5 (TextureRect)            ← 能量球分层贴图
│   └── RotationLayers (Control)
│       └── Layer2~3 (TextureRect)
├── BurstFront (CPUParticles2D)           ← 旧 burst 特效，保留
├── MyEnergyVfxFront (SNParticlesContainer) ← 能量增加时前景 burst
│   └── Burst (GPUParticles2D)
└── Label (Label, SMegaLabel)             ← 能量数值
```

**关键节点**：
- `MyEnergyVfxBack` / `MyEnergyVfxFront` 必须设置 `unique_name_in_owner = true`，以便 `%MyEnergyVfxBack` / `%MyEnergyVfxFront` 路径命中
- 位置均设为 `(64, 64)`，即能量球中心

## 调用链

```
能量增加
  └─ NEnergyCounter.OnEnergyChanged(old, new)
       ├─ [IL Patch] GetNode → GetNodeOrNull（初始化阶段已完成）
       ├─ 父类逻辑：_backVfx?.Restart()  // null，无效果
       └─ [Postfix Patch]
            └─ if (SKEnergyCounter)
                 ├─ _myBackVfx?.Restart()  → Glow 粒子重播
                 └─ _myFrontVfx?.Restart() → Burst 粒子重播
```

---

## 注意事项

1. **字段可见性**：`SKEnergyCounter._myBackVfx` / `_myFrontVfx` 必须是 `public`，因为 Patch 在 `ShoujoKagekiAijoKaren.src.Core.Patches` 命名空间中，无法访问 `private` / `protected` 字段
2. **场景节点名**：`MyEnergyVfxBack` / `MyEnergyVfxFront` 必须与 `SKEnergyCounter._Ready()` 中的 `%` 路径匹配，且必须设置 `unique_name_in_owner = true`
3. **IL Patch 仅针对 `NEnergyCounter._Ready()`**：不会影响其他类中正常的 `GetNode` 调用
4. **其他角色不受影响**：`NEnergyCounterOnEnergyChangedPatch` 中检查了 `is SKEnergyCounter`，其他角色走正常父类逻辑
