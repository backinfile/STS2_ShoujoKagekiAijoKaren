# Godot 视频特效接入经验

> 主题：在 STS2 Godot / C# mod 中接入一次性播放的视频特效  
> 参考实现：`src/Core/PromisePileSystem/Vfx/NKarenLastWordVideoVfx.cs`  
> 最近整理：2026-04-29

---

## 最重要的约定

**`.ogv` 视频文件由用户自己导出和转换。Codex 只负责代码接入、播放逻辑、层级、遮罩、音效和降级处理。**

不要让 Codex 自动批量转码视频，尤其不要在效果已经能播放后继续改视频格式。之前遇到过错误格式导致画面雪花、乱码的问题，最终以用户自行转换后的 `.ogv` 为准。

当前约定路径示例：

```text
res://ShoujoKagekiAijoKaren/video/last_word.ogv
D:\Godot\Proj\STS2_ShoujoKagekiAijoKaren\ShoujoKagekiAijoKaren\video\last_word.ogv
```

## 推荐流程

1. 用户先准备并导出 `.ogv`。
2. 将视频放到 `ShoujoKagekiAijoKaren/video/`。
3. 代码中用 `ResourceLoader.Load<VideoStream>(VideoPath)` 加载。
4. 加载失败时返回 `false`，让卡牌逻辑走普通 VFX fallback。
5. 视频特效节点挂到 `NRun.Instance.GlobalUi`，没有时再退到 `NCombatRoom.Instance`。
6. 播放前按需要停音乐，只保留专属 SFX。

## Godot 播放视频的基础写法

核心节点是 `VideoStreamPlayer`：

```csharp
_player = new VideoStreamPlayer
{
    Stream = _stream,
    Expand = true,
    Loop = false,
    Volume = 0f,
    Visible = false,
    MouseFilter = MouseFilterEnum.Ignore
};
```

注意点：

- `Expand = true` 让播放器按设置的 `Size` 显示。
- 通过 `Size` 和 `Position` 自己做居中和宽高比。
- 如果视频自带音轨但不想播放，设 `Volume = 0f`。
- 播放器可以先 `Visible = false`，等黑屏或延迟准备好后再 `Play()`。

## 层级经验

这次 Last Word 的目标层级是从前到后：

```text
视频（无黑边） -> T 字 -> 黑屏
```

实现时不要只相信 `ZIndex`，还要控制子节点添加顺序。实际更稳的顺序是：

1. 先 `AddChild(_black)`
2. 再添加 T 字 VFX
3. 最后 `AddChild(_player)`
4. 额外执行 `MoveChild(_player, GetChildCount() - 1)`

同时保留相对 Z 值作为双保险：

```csharp
private const int BlackZIndex = 0;
private const int LetterZIndex = 10;
private const int VideoZIndex = 20;
```

全屏 overlay 自身可以用高一点的绝对 Z，但不要超过 Godot canvas item 限制。当前安全值：

```csharp
private const int OverlayZIndex = 4095;
```

之前使用过过大的 Z 值，会触发 Godot 的 canvas item Z 上限报错。

## 画面布局

视频不要用黑边去填充画面，而是按视频真实宽高比缩放并居中：

```csharp
private const float VideoAspect = 1280f / 544f;

float videoHeight = viewportSize.Y * 0.7f;
float videoWidth = videoHeight * VideoAspect;
if (videoWidth > viewportSize.X * 0.96f)
{
    videoWidth = viewportSize.X * 0.96f;
    videoHeight = videoWidth / VideoAspect;
}

_player.Size = new Vector2(videoWidth, videoHeight);
_player.Position = (viewportSize - _player.Size) * 0.5f;
```

如果用户说“视频位置偏了”或“黑边不对”，优先检查：

- `VideoAspect` 是否和用户导出的 `.ogv` 一致
- `_player.Size`
- `_player.Position`
- 是否误用了全屏拉伸

## 黑屏与淡出

黑屏用 `ColorRect`，铺满全屏：

```csharp
_black = new ColorRect
{
    Color = new Color(0f, 0f, 0f, 0f),
    MouseFilter = MouseFilterEnum.Ignore
};
_black.SetAnchorsPreset(LayoutPreset.FullRect);
```

可以先透明，等视频开始时再设为黑色：

```csharp
_black.Color = Colors.Black;
_player.Visible = true;
_player.Play();
```

结束时淡出整个 overlay：

```csharp
float alpha = 1f - Mathf.Clamp(_fadeTimer / FadeOutSeconds, 0f, 1f);
Modulate = new Color(1f, 1f, 1f, alpha);
```

## fallback 很重要

视频播放特效建议返回 `bool`：

```csharp
public static bool Play()
{
    var stream = ResourceLoader.Load<VideoStream>(VideoPath);
    if (stream == null)
        return false;

    var parent = (Control?)NRun.Instance?.GlobalUi ?? NCombatRoom.Instance;
    if (parent == null)
        return false;

    parent.AddChildSafely(new NKarenLastWordVideoVfx(stream));
    return true;
}
```

卡牌逻辑里这样接：

```csharp
bool playedVideo = NKarenLastWordVideoVfx.Play();
if (!playedVideo)
    NKarenLastWordVfx.Play();
await Cmd.Wait(playedVideo ? 6.7f : 0.7f);
```

这样即使用户没有放 `.ogv`，或者 Godot 没加载成功，也不会导致卡牌完全没有表现。

## 音频处理

Last Word 这种大演出应先停音乐：

```csharp
KarenFormMusicManager.StopForCutscene();
KarenAudioManager.PlaySfx(KarenSfx.LastWord, volume: 1f);
```

`StopForCutscene()` 应同时处理：

- 本 mod 自己的循环 BGM
- 游戏原生 `NRunMusicController.Instance?.StopMusic()`
- 不要自动恢复 BGM，避免 cutscene 中途音乐又回来

## 常见问题

**视频雪花/乱码**

通常是 `.ogv` 导出格式不对。不要在代码里绕这个问题，交给用户重新导出 `.ogv`。

**T 字盖在视频前面**

检查添加顺序和 `MoveChild(_player, GetChildCount() - 1)`。视频播放器必须最后绘制。

**T 字看不见**

检查 T 字节点是否被视频盖住。如果目标层级是“视频在最前”，那 T 字本来就只会在视频外或透明区域可见；如果要看见更多 T 字，需要调视频素材透明度或改层级目标。

**Godot ZIndex 报错**

不要用 `10000` 这类过大值。全屏 overlay 当前使用 `4095`。

**视频时长和卡牌等待不一致**

优先按素材实际长度调整 `await Cmd.Wait(...)`，避免伤害结算提前或拖太久。

## 本次经验结论

- `.ogv` 导出必须由用户自己完成。
- 代码只做播放、布局、层级、fallback。
- 全屏视频特效应挂到 `GlobalUi`。
- 层级要同时控制 `ZIndex` 和 child order。
- 大演出开始前要停 BGM。
- 视频素材有问题时，不要用代码修画面格式，重新导出视频。
