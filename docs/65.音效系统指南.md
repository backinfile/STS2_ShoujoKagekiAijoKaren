# STS2 音效系统指南

## 一、游戏本体音效架构

### 1.1 核心技术栈

游戏使用 **FMOD** 作为音频中间件（通过 Godot 的 FMOD 插件集成），所有音效事件以 `event:/` 路径格式管理。FMOD Bank 文件位于游戏本体的 `banks/desktop/` 目录下：

```
banks/desktop/
  Master.bank / Master.strings.bank  —— 主银行（事件路径索引）
  sfx.bank                             —— 通用音效
  ambience.bank                        —— 环境音
  act1_a1.bank / act1_a2.bank / ...   —— 各章节音乐
  temp_sfx.bank                        —— 临时/调试音效
```

> **关键认知**：Mod 无法直接将自己的 `.bank` 文件注入到游戏 FMOD 系统中，因为 FMOD 在银行初始化后无法动态加载新银行。因此 Mod 添加音效需走 **Godot 原生音频** 路线。

### 1.2 核心类一览

| 类名 | 文件路径 | 作用 |
|------|---------|------|
| `NAudioManager` | `src/Core/Nodes/Audio/NAudioManager.cs` | 游戏主音频管理器，封装 FMOD 调用 |
| `SfxCmd` | `src/Core/Commands/SfxCmd.cs` | 播放音效的命令接口（推荐调用入口） |
| `FmodSfx` | `src/Core/Audio/FmodSfx.cs` | 常量定义，汇总了所有通用 FMOD 事件路径 |
| `NDebugAudioManager` | `src/Core/Audio/Debug/NDebugAudioManager.cs` | Debug 模式音频播放器（Godot 原生 AudioStreamPlayer） |
| `TmpSfx` | `src/Core/Audio/Debug/TmpSfx.cs` | Debug 音效文件名常量 |
| `AudioManagerProxy` | `src/gdscript/audio_manager_proxy.gd` | GDScript FMOD 代理节点 |

### 1.3 播放音效的三种方式

#### 方式 A：通过 SfxCmd（推荐，用于原版 FMOD 音效）

```csharp
using MegaCrit.Sts2.Core.Commands;

// 播放简单音效（一发声）
SfxCmd.Play(FmodSfx.block);        // 获得格挡
SfxCmd.Play(FmodSfx.buff);         // 获得增益
SfxCmd.Play(FmodSfx.debuff);       // 获得减益
SfxCmd.Play(FmodSfx.heal);         // 治疗
SfxCmd.Play(FmodSfx.gainEnergy);   // 获得能量
SfxCmd.Play(FmodSfx.enchant);      // 附魔
SfxCmd.Play(FmodSfx.uiClick);      // UI 点击
SfxCmd.Play(FmodSfx.uiHover);      // UI 悬停
SfxCmd.Play(FmodSfx.fire);         // 火焰攻击

// 带参数播放
SfxCmd.Play("event:/sfx/xxx", "参数名", 参数值, volume: 0.8f);

// 播放循环音效
SfxCmd.PlayLoop("event:/music/xxx", usesLoopParam: true);
SfxCmd.StopLoop("event:/music/xxx");
SfxCmd.SetParam("event:/music/xxx", "参数名", 值);
```

`SfxCmd` 会自动处理两个边界条件：
- `NonInteractiveMode.IsActive` —— 非交互模式（如测试）时不播放
- `CombatManager.Instance.IsEnding` —— 战斗结束时停止播放伤害/死亡音效

#### 方式 B：通过 NAudioManager（底层访问）

```csharp
using MegaCrit.Sts2.Core.Nodes.Audio;

NAudioManager.Instance?.PlayOneShot(FmodSfx.block);
NAudioManager.Instance?.PlayOneShot("event:/sfx/xxx", volume: 1f);
NAudioManager.Instance?.PlayOneShot("event:/sfx/xxx", 
    new Dictionary<string, float> { { "Intensity", 2f } }, volume: 1f);
```

#### 方式 C：通过 NDebugAudioManager（Debug 模式 / Godot 原生音频）

```csharp
using MegaCrit.Sts2.Core.Audio.Debug;

// 播放 debug_audio/ 目录下的 mp3/wav 文件
NDebugAudioManager.Instance?.Play("slash_attack.mp3");
NDebugAudioManager.Instance?.Play("heavy_attack.mp3", volume: 0.8f, 
    variance: PitchVariance.Small);  // 小幅随机音高变化
```

Debug 音频系统使用 Godot 原生的 `AudioStreamPlayer` 播放，**不依赖 FMOD**。文件路径通过 `TmpSfx.GetPath()` 解析为 `res://debug_audio/xxx.mp3`。

## 二、角色音效规范

### 2.1 角色音效的自动生成

在 `CharacterModel` 中，角色的攻击/施法/死亡音效按角色 ID 自动生成路径：

```csharp
// CharacterModel.cs
public string AttackSfx => $"event:/sfx/characters/{Id.Entry.ToLowerInvariant()}/{Id.Entry.ToLowerInvariant()}_attack";
public string CastSfx   => $"event:/sfx/characters/{Id.Entry.ToLowerInvariant()}/{Id.Entry.ToLowerInvariant()}_cast";
public string DeathSfx  => $"event:/sfx/characters/{Id.Entry.ToLowerInvariant()}/{Id.Entry.ToLowerInvariant()}_die";
public string CharacterSelectSfx => $"event:/sfx/characters/{Id.Entry.ToLowerInvariant()}/{Id.Entry.ToLowerInvariant()}_select";
public string CharacterTransitionSfx => "event:/sfx/ui/wipe_" + Id.Entry.ToLowerInvariant();
```

例如 Ironclad（ID = `"Ironclad"`）：
- 攻击音效 = `event:/sfx/characters/ironclad/ironclad_attack`
- 死亡音效 = `event:/sfx/characters/ironclad/ironclad_die`

### 2.2 自定义角色音效

Karen 的角色 ID 是 `"KAREN"`，因此自动生成路径为：
- `event:/sfx/characters/karen/karen_attack`
- `event:/sfx/characters/karen/karen_die`
- `event:/sfx/characters/karen/karen_cast`

**但这些事件需要在 FMOD Bank 中存在才能播放**。由于 Mod 无法动态注入 FMOD Bank，Karen 的角色音效需要通过其他方式实现。

## 四、攻击命令中的音效

在构建攻击命令时，可通过 `.WithAttackerFx()` 附加攻击音效：

```csharp
// 方式 1：使用角色默认攻击音效（推荐）
AttackBuilder.Attack(target, info)
    .WithAttackerFx(null, AttackSfx);   // 传入 null 为 VFX，AttackSfx 为音效

// 方式 2：使用自定义音效路径
AttackBuilder.Attack(target, info)
    .WithAttackerFx(null, "event:/sfx/enemy/enemy_attacks/xxx/xxx_attack");

// 方式 3：在攻击命令构建器中设置 HitSfx
new AttackCommand(...)
    .WithHitSfx("event:/sfx/xxx", tmpSfx: "debug_audio_name.mp3");
```

`AttackCommand` 执行时会自动播放 `HitSfx`：

```csharp
if (HitSfx != null)
    SfxCmd.Play(HitSfx);
else if (TmpHitSfx != null)
    NDebugAudioManager.Instance?.Play(TmpHitSfx);
```

## 六、Mod 中添加音效的方法

### 6.1 方案对比

| 方案 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| **A. 复用原版 FMOD 事件** | 直接调用 `SfxCmd.Play(FmodSfx.xxx)` | 与原版音量设置同步，零额外文件 | 只能使用已有音效，无法自定义 | 通用音效（打击/格挡/治疗等） |
| **B. Godot 原生 AudioStreamPlayer** | 在 Mod 中创建 `AudioStreamPlayer` 节点播放音频文件 | 完全自定义，文件格式自由 | 音量不跟随游戏 SFX 设置（需自己处理） | Mod 专属音效 |
| **C. Debug 音频系统** | 复用 `NDebugAudioManager` | 音量跟随游戏设置（Debug 音量） | 仅 Debug 模式可用，有播放池限制 | 开发期临时音效 |

### 6.2 推荐方案：Godot 原生音频（方案 B）

这是 Mod 添加自定义音效最可行的方式。

#### 步骤 1：准备音频文件

将音频文件（推荐 `.ogg` 格式，Godot 原生支持最好）放入 Mod 的插件目录：

```
ShoujoKagekiAijoKaren/
  audio/                      <-- 新建目录
    karen_attack.ogg
    karen_cast.ogg
    karen_die.ogg
    karen_skill_shine.ogg
    ui_promise_pile_open.ogg
    ...
```

确保音频文件在 Godot 编辑器中被导入（会在同目录生成 `.ogg.import` 文件）。

#### 步骤 2：创建音效管理器

```csharp
// src/Core/Audio/KarenAudioManager.cs
using Godot;

namespace ShoujoKagekiAijoKaren.src.Core.Audio;

public static class KarenAudioManager
{
    // 缓存已加载的音频流
    private static readonly System.Collections.Generic.Dictionary<string, AudioStream> _cache = new();

    /// <summary>
    /// 播放 Mod 音频文件（短音效）
    /// </summary>
    public static void Play(string audioFileName, float volume = 1f, float pitchScale = 1f)
    {
        var stream = GetStream(audioFileName);
        if (stream == null) return;

        var player = new AudioStreamPlayer
        {
            Stream = stream,
            VolumeLinear = volume,
            PitchScale = pitchScale,
            Bus = "SFX"   // 使用 Godot 的 SFX 总线
        };

        // 创建临时节点并播放
        var tree = Engine.GetMainLoop() as SceneTree;
        tree?.Root.AddChild(player);
        player.Play();

        // 播放完毕后自动释放
        player.Finished += () =>
        {
            player.QueueFree();
        };
    }

    /// <summary>
    /// 加载音频流（带缓存）
    /// </summary>
    private static AudioStream GetStream(string fileName)
    {
        if (_cache.TryGetValue(fileName, out var cached))
            return cached;

        var path = $"res://ShoujoKagekiAijoKaren/audio/{fileName}";
        var stream = GD.Load<AudioStream>(path);
        if (stream == null)
        {
            GD.PrintErr($"[KarenAudioManager] Failed to load audio: {path}");
            return null;
        }

        _cache[fileName] = stream;
        return stream;
    }

    /// <summary>
    /// 清除缓存（热重载时使用）
    /// </summary>
    public static void ClearCache() => _cache.Clear();
}
```

#### 步骤 3：在卡牌/能力中使用

```csharp
// 在卡牌打出时播放音效
protected override async Task OnPlay(PlayerChoiceContext choiceContext, CardPlay cardPlay)
{
    // 播放自定义音效
    KarenAudioManager.Play("karen_skill_shine.ogg", volume: 0.9f);

    // ... 原有逻辑
    await AttackBuilder.Attack(choiceContext.GetTarget(), ...)
        .WithAttackerFx(null, "event:/sfx/characters/attack_fire")  // 可混用原版 FMOD 音效
        .Submit();
}
```

#### 步骤 4：在角色动画中播放音效

如果 Karen 有自定义攻击动画，在动画回调或攻击命令中播放：

```csharp
// 覆盖 Karen 角色的攻击音效（通过 AttackBuilder）
AttackBuilder.Attack(target, info)
    .WithAttackerFx(karenAttackVfx, null);  // VFX 用自定义，音效在 OnPlay 中手动播放

// 或者在攻击命中时播放
await AttackBuilder.Attack(target, info)
    .WithOnHitCallback(async (ctx, hit) =>
    {
        KarenAudioManager.Play("karen_attack_hit.ogg");
    })
    .Submit();
```

### 7.1 游戏本体的音量控制

游戏通过 FMOD Bus 控制音量：

```
bus:/master           —— 主音量
bus:/master/sfx       —— SFX 音量
bus:/master/music     —— BGM 音量
bus:/master/ambience  —— 环境音音量
```

**简化方案**：将 Mod 音效也输出到 Godot 的 `SFX` 总线，并在设置变更时同步调整：

## 八、常见问题

### Q1：可以给 Karen 添加角色专属 FMOD 音效吗？

**不能直接添加**。FMOD Bank 在游戏启动时加载，Mod 无法在运行时注入新的 `.bank` 文件。只能通过 Godot 原生音频播放自定义音效。

### Q2：如何在攻击动画中精确同步音效？

```csharp
// 方式 1：在 AttackBuilder 中使用 WithAttackerFx 的音效参数
// 这会在攻击动画开始时播放
AttackBuilder.Attack(target, info)
    .WithAttackerFx(vfxNode, FmodSfx.fire);  // 原版 FMOD

// 方式 2：在 OnPlay 中手动控制时间
KarenAudioManager.Play("karen_attack.ogg");  // 立即播放
await Task.Delay(200);  // 延迟 200ms 后执行实际伤害

// 方式 3：在动画回调中播放（需要自定义动画系统）
```

### Q3：循环音效如何停止？

如果使用 Godot 原生音频：

```csharp
private AudioStreamPlayer _loopPlayer;

public void StartLoop()
{
    _loopPlayer = new AudioStreamPlayer
    {
        Stream = GetStream("ambient_loop.ogg"),
        Bus = "SFX"
    };
    Engine.GetMainLoop().CallDeferred("add_child", _loopPlayer);
    _loopPlayer.Play();
}

public void StopLoop()
{
    _loopPlayer?.Stop();
    _loopPlayer?.QueueFree();
    _loopPlayer = null;
}
```

### Q4：音效文件格式推荐？

| 格式 | 推荐度 | 说明 |
|------|--------|------|
| `.ogg` | ★★★★★ | Godot 原生支持最好，压缩率高，无专利问题 |
| `.wav` | ★★★☆☆ | 无损，但文件大，适合短音效 |
| `.mp3` | ★★★☆☆ | 支持但导入稍慢，有专利问题 |
