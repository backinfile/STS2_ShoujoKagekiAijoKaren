# Karen核心系统

> 主题：Karen 专属机制的系统说明  
> 适用范围：Shine（闪耀）、PromisePile（约定牌堆）、GlobalMove（全局移牌）、ExtraReplay（额外重播）、DisableRelic（禁用遗物）及相关扩展  
> 阅读建议：系统改动优先先看本篇，再回到 `03` 和 `09`

---

## 包含哪些系统

Karen 当前最核心的自定义系统包括：

- Shine（闪耀）
- PromisePile（约定牌堆）
- GlobalMove（全局移牌）
- ExtraReplay（额外重播）
- DisableRelic（禁用遗物）
- 奖励扩展
- 存档扩展
- 角色资源与能量相关表现

## Shine（闪耀）

核心文件：

- `src/Core/ShineSystem/ShineExtension.cs`
- `src/Core/ShineSystem/ShinePileManager.cs`
- `src/Core/ShineSystem/Patches/ShinePatch.cs`

作用：

- 给卡牌附加闪耀值。
- 在打牌流程中递减闪耀。
- 在闪耀耗尽时改变卡牌的后续处理。

常见 API：

- `IsShineCard()`
- `HasShine()`
- `GetShineValue()`
- `GetShineMaxValue()`
- `AddShineMax(...)`
- `SetShineCurrent(...)`
- `RestoreShineToMax()`
- `SetEnterShinePileAfterPlay(...)`

## PromisePile（约定牌堆）

核心文件：

- `src/Core/Commands/PromisePileCmd.cs`
- `src/Core/PromisePileSystem/PromisePileManager.cs`
- `src/Core/PromisePileSystem/PromisePileHooks.cs`

作用：

- 维护 Karen 的约定牌堆。
- 支持加入、抽取、切换、清空和从其他牌堆转入。
- 支持特殊模式。

常见入口：

- `Add`
- `Draw`
- `AddFromHand`
- `AddFromDiscard`
- `AddFromDraw`
- `SwitchHand`
- `DiscardAll`

常见模式：

- 普通模式
- `Void`
- `InfiniteReinforcement`

## GlobalMove（全局移牌）

核心文件：

- `src/Core/GlobalMoveSystem/GlobalMoveSystem.cs`
- `src/Core/GlobalMoveSystem/Patches/GlobalMovePatch.cs`

作用：

- 监听卡牌在牌堆间的移动。
- 把一次打牌过程中的多段移动整合为更稳定的“最终移动”语义。

卡牌侧入口：

- `OnGlobalMove(PileType from, PileType to, AbstractModel? source)`

## ExtraReplay（额外重播）

核心文件：

- `src/Core/ExtraReplaySystem/ExtraReplayExtension.cs`
- `src/Core/ExtraReplaySystem/Patches/ExtraReplayPatch.cs`

作用：

- 给卡牌附加“下次打出时额外重播若干次”的状态。

## DisableRelic（禁用遗物）

核心文件：

- `src/Core/DisableRelicSystem/DisableRelicVar.cs`
- `src/Core/DisableRelicSystem/Patches/DisableRelicViewPatch.cs`

作用：

- 让部分卡牌以禁用遗物数量为条件或数值来源。
- 提供展示层所需的 DynamicVar（动态变量）和视图补丁。

## 相关扩展

除了上述主系统，本项目还有和 Karen 玩法紧密关联的扩展：

- 奖励流程扩展
- 存档字段扩展
- 角色资源与能量显示扩展

这些内容分别和 `07`、`09`、`11` 有交叉。

## 系统改动时的检查点

1. 是否影响卡牌基类和现有 Hook（钩子）。
2. 是否影响结果牌堆或牌堆可视化。
3. 是否需要补联机同步与选择流程兼容。
4. 是否需要补存档、读档或运行期字段迁移。
5. 是否需要更新文案、关键字和 UI 展示。
