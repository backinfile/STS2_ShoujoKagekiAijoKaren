# UI、音频与表现系统

> 主题：UI、VFX、SFX 与 Karen 专属表现层  
> 适用范围：战斗表现、音频调用、能量计数器和界面接入点  
> 阅读建议：只有在做表现或排查视图问题时再深入本篇

---

## UI 与 VFX

项目表现层主要依赖：

- `VfxCmd`
- Godot 场景节点
- `NCombatRoom` 中的各类容器

常见内容包括：

- 战斗特效
- 牌堆查看界面
- 角色特效节点
- 顶栏与能量相关 UI

## 音频

音频路线包含：

- 原生 FMOD 路线
- Godot 原生音频补充路线

项目内最常见的音频调用入口：

- `SfxCmd`
- `KarenAudioManager`
- `KarenSfx`

## Karen 能量计数器

相关文件：

- `src/Core/Node/SKEnergyCounter.cs`
- `src/Core/Patches/NEnergyCounterReadyPatch.cs`
- `src/Core/Patches/NEnergyCounterOnEnergyChangedPatch.cs`
- `scenes/combat/energy_counters/karen_energy_counter.tscn`

用途：

- 定制 Karen 的能量显示。
- 桥接原生能量变化回调。
- 播放 Karen 专属能量特效。

## 排查表现问题时优先看什么

1. 是否已有对应 `Cmd`。
2. 视图节点是在 Godot 场景里，还是通过 Patch 挂进去。
3. 是否涉及卡牌扩展状态的额外 UI 展示。
4. 是否还需要同步处理音效和关键字提示。
