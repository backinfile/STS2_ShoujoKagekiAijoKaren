# UI、音频与表现系统

> 主题：UI、VFX、SFX 与 Karen 专属表现层  
> 适用范围：战斗表现、音频调用、能量计数器和界面接入点  
> 阅读建议：只有在做表现或排查视图问题时再深入本篇

---

## 表现层结构

Karen 的表现层不是单个目录能说完的，而是分散在：

- `src/Core/Patches/`
- `src/Core/Audio/`
- `src/Core/Node/`
- `scenes/`
- `images/`
- `materials/`
- `localization/`

当前代码说明 Karen 的表现实现思路是：

- 能复用原版界面的地方，优先用补丁接线
- 需要自定义节点的地方，再挂自己的 Godot 节点
- 音效走单独的 `KarenAudioManager`

## HoverTip 与关键字提示

相关文件：

- `src/Core/Patches/CardHoverTipsPatch.cs`
- `src/Core/Patches/CardKeywordInlinePatch.cs`

### `CardHoverTipsPatch`

这个补丁给 `CardModel.HoverTips` 做了后处理，根据卡牌标签和状态动态追加提示。

当前已经接了这些关键词：

- `KAREN_SHINE`
- `KAREN_PROMISE_PILE`
- `KAREN_SHINE_CARD_REWARD`
- `KAREN_TMP_STRENGTH`
- `KAREN_RETAIN_TMP_STRENGTH`
- `KAREN_DISABLE_RELIC`
- `KAREN_TOKEN_CARD`

它的特点是：

- 不要求卡牌手工一张张写 HoverTip 列表
- 只要标签和状态接对，提示就能自动出现

### `CardKeywordInlinePatch`

这个补丁处理的是描述文本行内显示问题。

当前已处理的一个具体场景是：

- 当 `Innate` 和 `Ethereal` 同时出现在同一张 Karen 卡上时
- 去掉 badge 之间的换行
- 让它们在同一行显示

这说明项目已经开始关注“文本排版层”的细节，而不只是规则正确。

## Karen 能量计数器

相关文件：

- `src/Core/Node/SNEnergyCounter.cs`
- `src/Core/Patches/NEnergyCounterReadyPatch.cs`
- `src/Core/Patches/NEnergyCounterOnEnergyChangedPatch.cs`
- `scenes/combat/energy_counters/karen_energy_counter.tscn`

### 自定义节点

`SNEnergyCounter` 继承自原版 `NEnergyCounter`，在 `_Ready()` 中会：

- 先调用 `base._Ready()`
- 用 `AccessTools.Field(...)` 把原版 `_backVfx` / `_frontVfx` 置空
- 读取自己的：
  - `%MyEnergyVfxBack`
  - `%MyEnergyVfxFront`

这说明 Karen 的能量特效不是简单覆盖原节点，而是显式替换原版特效引用。

### 兼容补丁

`NEnergyCounterReadyPatch` 通过 `Transpiler` 把 `_Ready()` 中的：

- `GetNode<T>()`

替换成：

- `GetNodeOrNull<T>()`

作用是避免 Karen 自定义能量场景缺少某些原版节点时直接报错。

`NEnergyCounterOnEnergyChangedPatch` 则在能量增加时：

- 判断实例是不是 `SNEnergyCounter`
- 如果是，则重启 `_myBackVfx` / `_myFrontVfx`

说明 Karen 自定义能量特效在“数值变化”这一层也接上了原版时机。

## 音频系统

相关文件：

- `src/Core/Audio/KarenAudioManager.cs`

### 当前能力

`KarenAudioManager` 当前支持：

- `Play(...)`
  播放一次性短音效，结束后自动释放 `AudioStreamPlayer`
- `PlayLoop(...)`
  播放循环音效，需要外部手动停止和释放
- `GetDuration(...)`
  获取音频时长
- `ClearCache()`
  清理缓存

### 设计特点

- 资源路径走 `res://ShoujoKagekiAijoKaren/audio/...`
- 内部缓存 `AudioStream`
- 统一挂到 Godot `SceneTree.Root`
- 总线固定使用 `SFX`

这样做的好处是：

- 音量自动跟随游戏 SFX 设置
- 不依赖 Godot 资源导入系统也能直接用字节加载

支持的格式包括：

- `.ogg`
- `.mp3`
- `.wav`

## PromisePile / Shine 的表现层联动

虽然规则在系统层，但 PromisePile 和 Shine 都已经影响到表现层：

- PromisePile 会更新星星数量显示
- PromisePile 能打开专门的牌堆查看界面
- Void 模式会覆盖抽牌堆图标
- Shine 耗尽时会播放专门的耗尽动画

这意味着改系统时不能只看规则代码，很多时候还要反查表现接线是否同步更新。

## 排查表现问题时的顺序

如果界面或特效异常，建议按这个顺序排查：

1. 规则层状态有没有真的更新。
2. 是否已有对应 `Cmd` 或 Manager 更新视图。
3. 节点是原版节点被补丁接线，还是自定义 Godot 节点。
4. 是否还需要同步处理 HoverTip、关键字、音效和图标。
