## STS2 原生架构

项目主要依托以下原生概念：

- `AbstractModel`
  模型基类。
- `CardModel`
  卡牌核心入口。
- `PowerModel`
  Buff / Debuff 与战斗钩子入口。
- `Hook`
  事件钩子总线。
- `ModelDb`
  模型注册中心。

KarenMod 的策略不是替换原生框架，而是在其上做扩展：

- 继承原生或项目基类。
- 使用 `SpireField` / `SavedSpireField` 附加状态。
- 用 Harmony 补丁改写局部流程。

## 模型层分工

- `KarenBaseCardModel : CardModel`
  Karen 卡牌公共基类，承接 `OnPlay` 与项目扩展钩子。
- `KarenDisableRelicBaseCardModel : KarenBaseCardModel`
  额外处理禁用遗物条件。
- `KarenBasePower : PowerModel`
  Karen Power（能力）的公共扩展入口。
- `Karen : CharacterModel`
  角色配置、起始卡组与资源颜色。

## 战斗主流程

```text
玩家打牌
 -> CardModel.OnPlayWrapper
 -> Hook.BeforeCardPlayed
 -> CardModel.OnPlay
 -> Power / 遗物 / 附魔 / 诅咒响应
 -> Hook.AfterCardPlayed
 -> 结果牌堆结算
 -> Hook.AfterCardChangedPiles
```

KarenMod 主要在这些位置插入自定义逻辑：

- `OnPlayWrapper`
  Shine（闪耀）值递减与相关处理。
- `Hook.ModifyCardPlayCount`
  ExtraReplay（额外重播）。
- `Hook.ModifyCardPlayResultPileTypeAndPosition`
  Shine（闪耀）耗尽后的结果牌堆重定向。
- `Hook.AfterCardChangedPiles`
  GlobalMove（全局移牌）与 PromisePile（约定牌堆）追踪。
