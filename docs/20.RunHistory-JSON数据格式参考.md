# RunHistory JSON 数据格式参考

用于分析对局数据的 JSON Schema 文档。Mod 通过 `RunHistoryUploadPatch` 在每局结束后将 `RunHistory` 对象序列化为 JSON POST 到 `http://localhost:8080`。

---

## 顶层结构：RunHistory

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `schema_version` | `int` | 存档 schema 版本号 |
| `platform_type` | `string` | 平台类型，如 `"Steam"` `"Null"` |
| `game_mode` | `string` | 游戏模式：`"Standard"` / `"Daily"` |
| `win` | `bool` | **是否胜利**（核心分析字段） |
| `seed` | `string` | 本局种子 |
| `start_time` | `long` | Unix 时间戳（秒），对局开始时间 |
| `run_time` | `float` | 对局持续时间（秒） |
| `ascension` | `int` | **进阶等级**（0~N） |
| `build_id` | `string` | 游戏版本，如 `"pre-v0.42"` |
| `was_abandoned` | `bool` | 是否主动放弃 |
| `killed_by_encounter` | `string` | 死亡时击杀你的遭遇战 ModelId（如 `"MonsterId"`），胜利时为 `"none"` |
| `killed_by_event` | `string` | 死亡时击杀你的事件 ModelId，胜利时为 `"none"` |
| `players` | `RunHistoryPlayer[]` | 玩家列表（单人模式长度为 1） |
| `acts` | `string[]` | 经过的 Act ModelId 列表 |
| `modifiers` | `SerializableModifier[]` | 本局生效的自定义规则修改器 |
| `map_point_history` | `MapPointHistoryEntry[][]` | **完整地图路线**，外层为 Act，内层为该 Act 的节点历史 |

---

## RunHistoryPlayer（玩家最终状态）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | `ulong` | 玩家 NetId |
| `character` | `string` | **角色 ModelId**，如 `"Ironclad"` `"Silent"` `"Karen"` |
| `deck` | `SerializableCard[]` | **最终卡组**（含初始牌、奖励牌、事件牌等） |
| `relics` | `SerializableRelic[]` | **最终遗物** |
| `potions` | `SerializablePotion[]` | **最终药水** |
| `max_potion_slot_count` | `int` | 最大药水槽数 |

---

## SerializableCard（卡牌）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | `string` | **卡牌 ModelId**，如 `"Defend"` `"Strike"` `"KarenShineStrike"` |
| `current_upgrade_level` | `int` | 升级次数（0=未升级，1=+1，2=+2...），未升级时可能省略 |
| `enchantment` | `SerializableEnchantment` | 附魔信息，无附魔时省略 |
| `props` | `SavedProperties` | 卡牌自定义状态数据，通常省略 |
| `floor_added_to_deck` | `int` | 获得该卡牌的楼层（Act 内计数），可能省略 |

### SerializableEnchantment（附魔）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | `string` | 附魔类型 ModelId，如 `"Heavy"` `"Deadly"` |
| `amount` | `int` | 附魔数值 |
| `props` | `SavedProperties` | 附魔自定义数据，可能省略 |

---

## SerializableRelic（遗物）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | `string` | **遗物 ModelId**，如 `"BurningBlood"` `"StageHeart"` |
| `props` | `SavedProperties` | 遗物自定义状态（如充能计数），可能省略 |
| `floor_added_to_deck` | `int` | 获得该遗物的楼层，可能省略 |

---

## SerializablePotion（药水）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | `string` | **药水 ModelId** |
| `slot_index` | `int` | 药水槽位索引 |

---

## SerializableModifier（规则修改器）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | `string` | 修改器 ModelId |
| `props` | `SavedProperties` | 修改器参数，可能省略 |

## MapPointHistoryEntry（地图节点历史）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `map_point_type` | `string` | 节点类型，见下表 |
| `rooms` | `MapPointRoomHistoryEntry[]` | 该节点包含的房间 |
| `player_stats` | `PlayerMapPointHistoryEntry[]` | 玩家经过该节点时的状态快照 |

### MapPointType 枚举值

| 值 | 含义 |
|----|------|
| `"Unassigned"` | 未分配 |
| `"Unknown"` | 未知 |
| `"Shop"` | 商店 |
| `"Treasure"` | 宝箱 |
| `"RestSite"` | 篝火休息点 |
| `"Monster"` | 普通怪物 |
| `"Elite"` | 精英 |
| `"Boss"` | Boss |
| `"Ancient"` | 远古（特殊） |

### MapPointRoomHistoryEntry（房间历史）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `room_type` | `string` | 房间类型，见下表 |
| `model_id` | `string` | 该房间/遭遇战的 ModelId（如怪物 ID、事件 ID） |
| `monster_ids` | `string[]` | 战斗中出现的怪物 ModelId 列表 |
| `turns_taken` | `int` | 该房间花费的回合数 |

### RoomType 枚举值

| 值 | 含义 |
|----|------|
| `"Unassigned"` | 未分配 |
| `"Monster"` | 普通怪物战 |
| `"Elite"` | 精英战 |
| `"Boss"` | Boss战 |
| `"Treasure"` | 宝箱 |
| `"Shop"` | 商店 |
| `"Event"` | 事件 |
| `"RestSite"` | 休息点 |
| `"Map"` | 地图节点（过渡） |

## SavedProperties（自定义状态数据）

某些卡牌/遗物/附魔会存储额外状态。结构为按类型分组的名值对列表：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `ints` | `{name: string, value: int}[]` | 整数属性 |
| `bools` | `{name: string, value: bool}[]` | 布尔属性 |
| `strings` | `{name: string, value: string}[]` | 字符串属性 |
| `int_arrays` | `{name: string, value: int[]}[]` | 整数数组属性 |
| `model_ids` | `{name: string, value: string}[]` | ModelId 属性 |
| `cards` | `{name: string, value: SerializableCard}[]` | 卡牌属性 |
| `card_arrays` | `{name: string, value: SerializableCard[]}[]` | 卡牌数组属性 |

> 注：`SavedProperties` 在绝大多数情况下为空或省略，只有带自定义计数器的遗物/卡牌才会出现。

## 数据分析建议

### 1. 卡牌胜率分析

```python
# 伪代码：统计某张卡出现在胜利/失败对局中的频率
wins_with_card = sum(1 for run in runs if run['win'] and any(c['id'] == 'CardName' for c in run['players'][0]['deck']))
total_with_card = sum(1 for run in runs if any(c['id'] == 'CardName' for c in run['players'][0]['deck']))
win_rate = wins_with_card / total_with_card
```

### 2. 遗物胜率分析

类似卡牌胜率，遍历 `players[0]['relics']` 按 `id` 分组统计胜率。

### 3. 卡组构筑强度（聚类分析）

- 提取每局的卡组 `id` 列表（忽略升级和附魔）
- 使用 Jaccard 相似度或 TF-IDF 对卡组做聚类
- 计算每个聚类的平均胜率

### 4. 进阶等级趋势

```python
# 按进阶等级分组统计胜率
for asc in range(0, 21):
    subset = [r for r in runs if r['ascension'] == asc]
    if subset:
        rate = sum(1 for r in subset if r['win']) / len(subset)
        print(f"Ascension {asc}: {rate:.2%}")
```

### 5. 死亡原因分析

```python
# 统计最常见的死亡遭遇战/事件
from collections import Counter
death_runs = [r for r in runs if not r['win']]
encounter_deaths = Counter(r['killed_by_encounter'] for r in death_runs if r['killed_by_encounter'] != 'none')
event_deaths = Counter(r['killed_by_event'] for r in death_runs if r['killed_by_event'] != 'none')
```

### 6. 路线与回合数分析

遍历 `map_point_history`：
- 计算每个 Act 的平均节点数
- 统计各种房间类型的出现频率
- 计算各类房间的平均 `turns_taken`
- 分析特定怪物/事件的死亡率

### 7. 角色胜率对比

```python
from collections import defaultdict
by_character = defaultdict(list)
for run in runs:
    char = run['players'][0]['character']
    by_character[char].append(run['win'])

for char, results in by_character.items():
    rate = sum(results) / len(results)
    print(f"{char}: {rate:.2%} ({len(results)} runs)")
```

### 8. 卡牌升级价值分析

对比同一张卡在不同升级等级下的胜率差异：

```python
card_levels = defaultdict(lambda: {'wins': 0, 'total': 0})
for run in runs:
    for card in run['players'][0]['deck']:
        key = (card['id'], card.get('current_upgrade_level', 0))
        card_levels[key]['total'] += 1
        if run['win']:
            card_levels[key]['wins'] += 1
```

## 注意事项

1. **ModelId 格式**：卡牌/遗物/角色 ID 是字符串，Karen Mod 的卡牌 ID 前缀通常为 `Karen`，如 `"KarenShineStrike"` `"KarenPromise"`
2. **附魔数据**：`enchantment` 字段在大多数卡牌上为 `null` 或省略，只有被附魔的卡牌才会出现
3. **药水数据**：`potions` 只记录最终剩余的药水，已使用的药水不会保留
4. **地图路线**：`map_point_history` 是嵌套数组，外层索引对应 Act（0=Act1），内层是该 Act 的所有节点历史
5. **多人模式**：`players` 列表长度可能大于 1，单人模式下 `players[0]` 即为本地玩家
