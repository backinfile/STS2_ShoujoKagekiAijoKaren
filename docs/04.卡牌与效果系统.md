# 卡牌与效果系统

> 主题：Karen 卡牌开发的主入口文档  
> 适用范围：卡牌基类、命令、DynamicVar（动态变量）、Power（能力）、Clone、附魔与诅咒  
> 阅读建议：写新卡时先看本篇，再按需要查 `05` 和 `10`

---

## 卡牌入口

Karen 卡牌主要继承：

- `KarenBaseCardModel : CardModel`
- `KarenDisableRelicBaseCardModel : KarenBaseCardModel`

最常见的覆写点：

- `CanonicalVars`
- `CanonicalTags`
- `CanonicalKeywords`
- `OnPlay(PlayerChoiceContext, CardPlay)`
- `OnUpgrade()`
- `GainsBlock`

`KarenBaseCardModel` 额外暴露的项目扩展钩子：

- `OnAddedToPromisePile()`
- `OnRemovedFromPromisePile()`
- `OnTurnEndInPromisePile()`
- `OnShineExhausted(...)`
- `OnGlobalMove(...)`
- `DoOption(...)`

## DynamicVar（动态变量）

项目里最常见的变量类型：

- `DamageVar`
- `BlockVar`
- `CardsVar`
- `PowerVar<T>`
- `EnergyVar`
- `DisableRelicVar`

常见用途：

- 驱动描述文本中的动态数值。
- 统一升级前后差异显示。
- 把系统扩展数值安全地暴露到 UI。

## 命令与效果表达

Karen 卡牌效果仍以原生命令体系为主，再叠加项目自定义命令。

常见原生命令方向：

- 伤害：`DamageCmd`
- 格挡：`BlockCmd`
- 抽牌 / 弃牌：相关牌堆命令
- 附加 Power（能力）：`CreatureCmd` 配合 `GainPower`

项目内常见自定义命令与入口：

- `PromisePileCmd`
- 各类 Option / 生成 / 切换相关命令

写卡时优先复用现有命令，而不是在 `OnPlay` 里直接堆杂逻辑。

## Power（能力）

Karen Power 主要基于：

- `PowerModel`
- `KarenBasePower`

使用 `KarenBasePower` 的好处：

- 更容易接入 PromisePile（约定牌堆）等项目扩展。
- 便于统一项目内的 Power（能力）行为。

如果效果本质上是持续状态、回合触发或 Hook（钩子）响应，优先考虑做成 Power（能力），而不是把状态塞进卡牌本体。

## Clone 与生成卡

项目内常见的复制与生成卡相关入口：

- `CreateClone`
- `CreateDupe`
- `RunState.CloneCard`
- `CardModelEx.CloneSafeForDeck`

需要特别注意：

- 复制时是否应保留动态字段。
- 生成卡是否应进入牌库、战斗内临时区域或结果牌堆。
- Shine（闪耀）、PromisePile（约定牌堆）、DisableRelic（禁用遗物）等扩展状态是否需要同步处理。

## 附魔与诅咒

除了卡牌与 Power（能力），本项目还会和这些系统交互：

- `Affliction`
- `Enchantment`

如果卡牌效果涉及额外状态层，先确认它应该落在：

- 卡牌瞬时效果
- Power（能力）
- 附魔 / 诅咒
- 项目专属系统扩展

## 开发检查表

新增一张卡时，通常至少要检查这些点：

1. 继承的基类是否正确。
2. `CanonicalVars` / `CanonicalKeywords` 是否齐全。
3. 文本是否和数值、升级逻辑一致。
4. 需要的命令是否已有现成实现。
5. 是否会与 Shine（闪耀）、PromisePile（约定牌堆）、GlobalMove（全局移牌）、DisableRelic（禁用遗物）等系统联动。
6. 生成卡、复制卡和联机选择流程是否安全。
