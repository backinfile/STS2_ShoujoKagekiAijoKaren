# Burn 特效复刻记录

> 主题：将 `D:\Github\STS_ShoujoKageki` 中 `Burn / 燃烧吧燃烧吧` 打出后的表现复刻到本项目 `KarenBurn`  
> 适用范围：`KarenBurn`、PromisePile Burn 模式、战斗角色持续 VFX  
> 最近整理：2026-04-26

---

## 目标

源项目中 `Burn` 卡牌打出后会进入燃烧状态，并持续播放类似 Watcher Wrath 姿态的红色粒子与光环效果。本项目已经有 `KarenBurn` 的规则效果：

- 进入 `PromisePileMode.Burn`
- 从约定牌堆抽出的牌会升级
- 被燃烧影响的牌打出时带有消耗语义

本次复刻目标是补齐打出 `KarenBurn` 后的持续战斗 VFX。

## 源项目链路

源项目位置：

- `D:\Github\STS_ShoujoKageki\src\main\java\ShoujoKageki\cards\bag\Burn.java`
- `D:\Github\STS_ShoujoKageki\src\main\java\ShoujoKageki\effects\BurnEffect.java`
- `D:\Github\STS_ShoujoKageki\src\main\java\ShoujoKageki\powers\BurnPower.java`

`Burn.java` 的核心行为：

```java
BagField.bagBurn.set(p, true);
BagField.bagUpgrade.set(p, true);
addToBot(new ApplyBagPowerAction());
AbstractDungeon.effectList.add(new BurnEffect());
```

`BurnEffect.java` 的核心行为：

```java
particleTimer -= Gdx.graphics.getDeltaTime();
if (particleTimer < 0.0F) {
    particleTimer = 0.05F;
    AbstractDungeon.effectsQueue.add(new WrathParticleEffect());
}

particleTimer2 -= Gdx.graphics.getDeltaTime();
if (particleTimer2 < 0.0F) {
    particleTimer2 = MathUtils.random(0.3F, 0.4F);
    AbstractDungeon.effectsQueue.add(new StanceAuraEffect("Wrath"));
}
```

结论：源项目的 `BurnEffect` 本身没有自定义火焰图片，也没有渲染逻辑；它只是持续向 STS 原版特效队列里塞两个原版效果：

- `WrathParticleEffect`
- `StanceAuraEffect("Wrath")`

## 原版参数参考

通过 `javap` 反编译 STS class 后得到关键参数。

### `WrathParticleEffect`

资源：

- `ImageMaster.GLOW_SPARK`

生成节奏：

- 由 `BurnEffect` 每 `0.05s` 生成一个

初始化参数：

- `duration`: `random(1.3f, 1.8f)`
- `scale`: `random(0.6f, 1.0f) * Settings.scale`
- `color`: `new Color(random(0.5f, 1.0f), 0, random(0, 0.2f), 0)`
- `rotation`: `random(-8f, 8f)`
- `renderBehind`: `randomBoolean(0.2f + scale - 0.5f)`

更新逻辑：

- 前半段 alpha 从 0 淡入到 1
- 后半段 alpha 从 1 淡出到 0
- 纵向速度每帧增加 `delta * 40 * Settings.scale`
- 渲染时 X 缩放约 `scale * 0.8`
- 渲染时 Y 缩放约 `0.1 + elapsed * 2 * scale`

### `StanceAuraEffect("Wrath")`

资源：

- `ImageMaster.EXHAUST_L`

生成节奏：

- 由 `BurnEffect` 每 `random(0.3f, 0.4f)` 生成一个

初始化参数：

- `duration`: `2f`
- `scale`: `random(2.7f, 2.5f) * Settings.scale`
- `color`: `new Color(random(0.6f, 0.7f), 0, random(0.1f, 0.2f), 0)`
- `rotation`: `random(360f)`
- 使用静态 `switcher` 交替前后层与旋转方向

更新逻辑：

- alpha 在 `0` 与 `0.3` 之间淡入淡出
- 按随机角速度旋转
- `switcher` 为真时在后层，旋转速度 `random(0, 40)`
- `switcher` 为假时在前层，旋转速度 `random(0, -40)`

## 本项目实现

相关文件：

- `src/Core/Models/Cards/neutral/KarenBurn.cs`
- `src/Core/PromisePileSystem/Vfx/KarenBurnVfxManager.cs`
- `src/Core/PromisePileSystem/Vfx/NKarenBurnVfx.cs`
- `src/Core/PromisePileSystem/PromisePileManager.cs`

### `KarenBurn.cs`

打出后进入 Burn 模式，并启动 VFX：

```csharp
await PromisePileManager.EnterMode(Owner, PromisePileMode.Burn);
KarenBurnVfxManager.Start(Owner);
```

### `KarenBurnVfxManager.cs`

职责：

- 按玩家缓存 `NKarenBurnVfx`
- 从 `NCombatRoom.Instance.GetCreatureNode(player.Creature)` 获取角色节点
- 将燃烧 VFX 挂到角色节点下
- 重复打出时复用并 `Restart()`
- 战斗结束或约定牌堆清理时 `Stop()`

### `NKarenBurnVfx.cs`

职责：

- 持续跟随 `_creatureNode.VfxSpawnPosition`
- 每 `0.05s` 添加 `NKarenWrathParticle`
- 每 `random(0.3, 0.4)s` 添加 `NKarenStanceAura`
- 使用加法混合 `CanvasItemMaterial.BlendModeEnum.Add`
- 复刻原版 alpha、旋转、生命周期与缩放变化

当前为了适配 Godot 坐标与 STS hitbox 差异，做了少量映射补偿：

- 横向散布范围使用 `PlayerWidth = 360f`
- `glowSpark` 的 X 缩放增加 `GodotWidthCompensation = 1.75f`

这不是玩法差异，而是坐标系差异。否则原版纵向拉伸逻辑在 Godot 角色坐标里会显得过细。

## 层级处理记录

中途出现过两个层级问题：

1. `ZIndex = 120` 太高，导致特效在卡牌选择界面和卡牌 UI 上方透出。
2. 全部改成负 `ZIndex` 后，又被角色或场景压住，看不到特效。

当前方案：

- 父节点 `ZAsRelative = true`
- 父节点 `ZIndex = 1`
- `WrathParticle` 子节点使用 `ZIndex = 0 / 1`
- `StanceAura` 子节点使用 `ZIndex = 0 / 1`

原则：

- 不要用超高 Z 层级盖 UI
- 不要用负层级把战斗 VFX 压到角色后方不可见
- 如果后续仍在牌库/选择界面穿透，优先考虑在相关 UI 打开时隐藏战斗 VFX，而不是继续硬调 Z

## 踩坑记录

- 源项目没有自带 `BurnEffect` 专用贴图，不能从 `D:\Github\STS_ShoujoKageki` 直接复制一张火焰图解决。
- `BurnPower84.png / BurnPower32.png` 是 Power 图标，不是打出后的 VFX 粒子资源。
- 原版 `vfx.png` 是 atlas，不应整张引入项目；只应裁剪使用到的 region。
- `combat/flame4/5/6` 和 `combat/verticalAura` 看起来像火焰，但不是 `BurnEffect` 真实调用链使用的资源。
- `WrathParticleEffect` 看似细长是原版设计；在本项目里需要做坐标映射补偿，否则视觉上会变成细线。
- VFX 应挂在战斗角色节点附近，而不是 UI 层。
