## 核心机制：闪耀系统（Shine System）
- 卡牌每次打出 Shine-1，归零从卡组移除；数据：`SpireField<CardModel, int>`
- 存档：GodotFileIo Patch 嵌入同一存档；恢复时机：`RunManager.SetUpSaved*` Postfix
- 耗尽动画：`NCard.FindOnTable` → Reparent 顶层 → 压扁变黑 Tween（支持 FastMode）
- 关键词提示文件名必须是 `card_keywords.json`，扁平格式 `"KEY.title"` / `"KEY.description"`
- **Shine显示格式**（`ShineGlobalPatch.cs`）：`label+coloredNumber+suffix`，键 `KAREN_SHINE_LABEL/SUFFIX`
  - 中文：`闪耀3。`（label 无尾随空格）；英文：`Shine 3.`（有空格）
  - 有 `CardKeyword.Exhaust` 时同行追加；否则换行
- 详细见 `docs/45.闪耀系统记忆.md`，动画见 `docs/53.闪耀耗尽动画指南.md`

## 核心系统：GlobalMoveSystem（全局卡牌移动监听）
- 文件：`src/Core/GlobalMoveSystem/`；原理：Prefix Patch `Hook.AfterCardChangedPiles`
- 接口：`GlobalMoveSystem.OnCardMoved += (card, from, to, source)`；`PileType.None` = 进/离战斗
- 注意：`Hook.AfterCardChangedPiles` 在 `MegaCrit.Sts2.Core.Hooks`；`CombatState` 在 `MegaCrit.Sts2.Core.Combat`
- **约定牌堆接入**：约定牌堆进/出也触发 `Hook.AfterCardChangedPiles`，`from/to` 用 `KarenCustomEnum.PromisePile`
  - 进入：`AddToPromisePile` 在 `pile.AddLast` 后 fire-and-forget 调 Hook；`GlobalMovePatch` 检测 `IsInPromisePile` 推断 newPile
  - 离开（Draw/DiscardAll）：`GlobalMovePatch.SuppressOnce.Add(card)` 抑制 `CardPileCmd.Add` 的错误触发，再手动调 Hook
  - `ClearPromisePile`（战斗结束）：不触发任何扳机，仅清理
  - 调 Hook 参数：`card.Owner.RunState` / `card.CombatState`（同 `CardPileCmd.cs` 内部写法）

## 核心系统：约定牌堆系统（PromisePileSystem）
- 文件：`PromisePileManager.cs` + `Commands/PromisePileCmd.cs` + `Patches/PromisePileCombatPatch.cs` + `PromisePileAnimator.cs` + `Patches/NCreatureClickPatch.cs`
- 数据：`SpireField<Player, LinkedList<CardModel>>`（FIFO），仅战斗中有效，战斗开始/结束自动清空
- 命令：`PromisePileCmd.Add(card)` 放入；`Draw(ctx, player, count)` 取出到手牌；`AddFromDiscard/AddFromDraw` 选牌入堆；`DiscardAll` 全部弃置
- 清空：`RemoveFromCurrentPile + AccessTools.Method("RemoveFromState")`
- 战斗开始 Patch：`Hook.BeforeCombatStart`；结束 Patch：`Player.AfterCombatEnd`
- 选牌 API：`CardSelectCmd.FromSimpleGrid(ctx, cards, player, prefs)`；`CardSelectorPrefs` 在 `MegaCrit.Sts2.Core.CardSelection`
- **动画**（`PromisePileAnimator.cs`）：
  - Add：`PlayAddAnimation(card)` 在 `RemoveFromCurrentPile()` 前调用；必须同时 RemoveChild NCard 和 holder（否则 `OnSelectModeSourceFinished` 会把卡加回手牌）
  - Draw：`await PlayDrawAnimationAsync(card, player)`；`NCard.Create` → GlobalUi → Tween Scale→1 → QueueFree → CardPileCmd.Add
  - 角色位置：`NCombatRoom.Instance?.GetCreatureNode(player.Creature)?.VfxSpawnPosition`
- **约定牌堆 Power**（`KarenPromisePilePower`）：
  - 继承 `FakeAmountPower`（`src/Core/Models/Powers/FakeAmountPower.cs`）显示自定义数值（不受 StackType 限制）
  - **Void 模式**（`EnterVoidMode()`）：交互重定向到抽牌堆，图标和描述切换
    - `PromisePileCmd` 所有操作自动重定向：`Add`→抽牌堆顶，`Draw`→从抽牌堆抽，`DiscardAll`→弃置所有抽牌堆
    - `AddFromDraw` 在 Void 模式下无效果；点击角色不打开界面
    - 图标自动切换：`karen_promise_pile_power.png` → `karen_promise_pile_power_void.png`
    - 描述键：`KAREN_PROMISE_PILE_POWER.voidTitle` / `voidDescription`（"我现在是世界上最空虚的人了"）
  - 只有 Karen 战斗开局自动获得（count=0）；其他玩家调用 Add/Draw 时动态创建
  - `SetCardNames(names[])`：FIFO 顺序展示队列内容（ExtraHoverTips，最多10张）
  - 本地化键：`KAREN_PROMISE_PILE_POWER.title/description/pileContents`（全大写下划线）
  - 图标：`images/powers/karen_promise_pile_power.png`
- **查看界面**（`NCreatureClickPatch`）：点击华恋角色模型打开；`PromisePileManager.ShowScreen(player)` 创建 `CardPile(PileType.None)` 快照 → `NCardPileScreen.ShowScreen`
  - 防误触：`NTargetManager.Instance.IsInSelection` + `LastTargetingFinishedFrame`
  - **打开条件：玩家身上有 `KarenPromisePilePower`**（不检查数量，空牌堆也直接打开界面）
  - 界面底部显示标题（`KAREN_PROMISE_PILE_INFO`）
  - 再次点击关闭界面

## 卡牌基类：KarenBaseCardModel
- 文件：`src/Core/Models/Cards/KarenBaseCardModel.cs`；命名空间：`ShoujoKagekiAijoKaren.src.Core.Models.Cards`
- **所有 Karen 卡牌必须继承此类**（`abstract class KarenBaseCardModel : CardModel`）
- 提供三个 `public virtual Task` 扳机，由管理器直接调用（无需手动订阅事件）：
  - `OnAddedToPromisePile()` — 卡牌被放入约定牌堆时（由 `PromisePileManager.AddToPromisePile` 调用）
  - `OnRemovedFromPromisePile()` — 卡牌离开约定牌堆时（Draw/DiscardAll/ClearPromisePile 均触发）
  - `OnShineExhausted()` — 闪耀耗尽时（由 `ShinePileManager.AddToShinePile` 调用，仅 `IsInProgress==true` 时触发）
- **边缘情况**：以最后1 Shine击杀敌人时 `IsInProgress==true` → 正常触发；存档恢复时 `IsInProgress==false` → 不触发
- **using 引用**：其他文件需加 `using ShoujoKagekiAijoKaren.src.Core.Models.Cards;`

## 卡牌复制（Clone）

- `CreateClone()` — 战斗中复制，卡必须在战斗牌堆；设置 `_cloneOf`；自动选 Scope
- `RunState.CloneCard(card)` — 战斗外复制（营地等），无校验；Owner 通过 MemberwiseClone 自动继承
- **底层机制**：`ClonePreservingMutability()` — 不可变对象返回自身引用，可变对象调用 `MutableClone()` 深拷贝
- **不自动复制**：`SpireField` 数据（Shine 值等），需手动同步
- **加入战斗堆**：`CreateClone()` + `CardPileCmd.AddGeneratedCardToCombat(clone, PileType.Hand, true)`
- **加入牌组（永久）**：`RunState.CloneCard()` + `CardPileCmd.Add(clone, PileType.Deck)` + `CardCmd.PreviewCardPileAdd(result, 1.2f, CardPreviewStyle.MessyLayout)`
- `PreviewCardPileAdd` 是 void，fire-and-forget，不需要 await；`CardPileCmd.Add` 本身无展示动画
- 复制进牌组的卡**无需加入 KarenCardPool**（不经过 ModelDb.AllCards）
- 详细见 `docs/03.卡牌复制指南.md`

### CreateClone() 自动复制的内容（基于 CardModel.DeepCloneFields）

| 属性 | 复制方式 | 是否复制 |
|------|----------|----------|
| `CurrentUpgradeLevel` | MemberwiseClone 浅拷贝 | ✅ 升级等级保留（+2克隆后仍是+2） |
| `Enchantment` | ClonePreservingMutability + EnchantInternal | ✅ 附魔状态和数值完整复制 |
| `Affliction` | ClonePreservingMutability + AfflictInternal | ✅ 诅咒状态和数值完整复制 |
| `Keywords` | HashSet 深拷贝 | ✅ 所有关键词保留 |
| `DynamicVars` | DynamicVars.Clone(this) | ✅ 动态变量完整复制 |
| `EnergyCost` | _energyCost?.Clone(this) | ✅ 费用状态保留 |
| `Shine值` | SpireField 外部存储 | ❌ **需手动同步** |

### 使用示例（复制带升级和附魔的卡牌）

```csharp
protected override async Task OnPlay(PlayerChoiceContext choiceContext, CardPlay cardPlay)
{
    CardModel selection = (await CardSelectCmd.FromHand(...)).FirstOrDefault();
    if (selection != null)
    {
        // CreateClone 自动复制升级状态和附魔
        CardModel clone = selection.CreateClone();

        // Karen Mod 需手动同步 Shine 值
        int shine = ShinePileManager.GetShine(selection);
        if (shine > 0) ShinePileManager.SetShine(clone, shine);

        await CardPileCmd.AddGeneratedCardToCombat(clone, PileType.Hand, addedByPlayer: true);
    }
}
```

## 卡牌开发常见 API 陷阱
- **StartingDeck 的卡必须同时加入 CardPool**：
  - `ModelDb.AllCards = AllCardPools里的所有卡 + AllCharacters.StartingDeck里的卡`（去重合并）
  - Preload 遍历 AllCards 时对每张卡访问 `.Pool`，若该卡不在任何 Pool 里，会访问 `MockCardPool.AllCardIds` → 崩溃（"You monster!"）
  - 凡是加入 `StartingDeck` 的卡，**必须同时加入对应 `CardPool.GenerateAllCards()`**
  - 例外：诅咒牌加入 `CurseCardPool`（通过 Patch），不加入 `KarenCardPool`
- **`TemporaryStrengthPower` 子类 + `PowerVar` 禁止混用**：
  - `PowerVar<T>` 的 `T` 只能是不含 `OriginModel` 的普通 Power（如 `StrengthPower`）
  - `TemporaryStrengthPower` 子类（如 `KarenChargeStrikeStrengthDownPower`）只在 `OnPlay` 的 `PowerCmd.Apply<T>()` 中使用
  - 原因：`PowerVar<TemporaryStrengthPower子类>` 初始化时会触发 `OriginModel`（该属性是 abstract 必须实现），`OriginModel` 调用 `ModelDb.Card<T>()` → `CardModel.Pool` → 遍历所有池含 `MockCardPool` → 崩溃
  - 正确写法：`CanonicalVars` 用 `PowerVar<StrengthPower>`，`ExtraHoverTips` 用 `FromPower<StrengthPower>()`，`OnPlay` 里 `PowerCmd.Apply<KarenXxxPower>(target, DynamicVars.Strength.BaseValue, ...)`
  - 本地化描述对应用 `{StrengthPower:diff()}`，不用自定义 Power 的类名
- **诅咒牌实现模式**：
  - 稀有度用 `CardRarity.Curse`（非 `Special`）
  - Tag/关键词用 `CanonicalKeywords` + `CardKeyword`（Unplayable/Ethereal/Eternal/Exhaust），非 `CanonicalTags`
  - 示例：困意诅咒 = `CardKeyword.Unplayable, CardKeyword.Ethereal, CardKeyword.Eternal`
  - **Karen 诅咒牌不放入 `KarenCardPool`，而是通过 Patch 注入原版 `CurseCardPool`**
  - Patch 文件：`src/Core/Patches/CurseCardPoolPatch.cs`（Postfix `GenerateAllCards`，用 `ref CardModel[] __result` 追加）
- **费用升级**：`EnergyCost.UpgradeBy(-1)`（非 `UpgradeCost(-1)`）
- **无限升级**：`public override int MaxUpgradeLevel => int.MaxValue;`
- **ValueProp 零值**：`(ValueProp)0`（无 `.None`）
- **Creature 血量**：`creature.CurrentHp`（非 `HP`）；`creature.IsAlive` = `CurrentHp > 0`
- **多段攻击**：`.WithHitCount(N)` 单目标；全体多轮用 for 循环
- **力量减少**：`PowerCmd.Apply<StrengthPower>(target, -amount, ...)` 传负数
- **无 PlayerChoiceContext 时**：fire-and-forget `_ = CreatureCmd.Heal(...)`
- **格挡悬浮提示**：打出时给格挡必须加 `public override bool GainsBlock => true;`
- **Power 效果 Tip**：override `ExtraHoverTips` 返回 `HoverTipFactory.FromPower<WeakPower>()` 等（命名空间 `MegaCrit.Sts2.Core.HoverTips`）
- **ShinePile 计数**：`ShinePileManager.GetShinePileCount(Owner)`
- **OnShineExhausted**：override `KarenBaseCardModel.OnShineExhausted()`（见下节），不再手动订阅事件

## DynamicVar / 本地化 JSON 陷阱
- **PowerVar 键名用全类名**：`{StrengthPower:diff()}`（非 `{Strength:diff()}`）
- **Var 语义**：段数/轮次 → `RepeatVar`；每条件加伤 → `ExtraDamageVar`；`CardsVar` 仅用于卡牌数量
- **能量显示**：`{Energy:energyIcons()}`，**不加**"点能量"文字
- **升级专属描述**：`{IfUpgraded:show:文本|}`，**不用** `[gray]...[/gray]`
- 完整参考：`docs/16.动态变量指南.md`

## Power 悬浮提示系统
- `PowerModel.Description` 是 key-based LocString，不支持原始字符串注入
- `PowerModel.ExtraHoverTips` 可 override；`HoverTip(LocString, string)` 的 string 接受原始字符串
- `HoverTips`/`SmartDescription` 均非虚，不可 override

### 基础结构
```csharp
// 能力牌
public sealed class KarenXxxCard() : KarenBaseCardModel(1, CardType.Power, CardRarity.Uncommon, TargetType.Self)
{
    protected override async Task OnPlay(PlayerChoiceContext ctx, CardPlay cardPlay)
    {
        await PowerCmd.Apply<KarenXxxPower>(Owner.Creature, amount, Owner.Creature, this);

// Power效果
public sealed class KarenXxxPower : PowerModel  // 或 KarenBasePower（约定牌堆相关）

    public override PowerType Type => PowerType.Buff;
    public override PowerStackType StackType => PowerStackType.Intensity;  // 或 Counter
```
